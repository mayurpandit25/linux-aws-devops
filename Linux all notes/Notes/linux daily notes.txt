HYPERVISOR:-
1.Hyervisor is a software that creates the space and runs the virtual machine. 
2.we use two operating system at a same time with the help of hypervisor.
   
TYPES OF HYPERVISOR:-
1.type-1 hypervisor (bare metal hypervisor)
2.type-2 hypervisor (hosted hypervisor)

TYPE-1 HYPERVISOR (bare-metal hypervisor) ====>>
1.Type-1 hypervisor is also called as bare-metal hypervisor. and also known as firmware.
2.its a hypervisor that directly runs on the host hardware. also known as hardware level virtualization
3.a guest os runs on another level above the hypervisor.
4.Type-1 hypervisor acts as their own operating system.
5.it has the direct access to the host hardware and resources.
6.more secure than type-2
7.example:- (Hyper-V, VMware ESXi)

Architecture:-

Hardware ========> Hypervisor =========> VM ========> O.S =========> centos/ubuntu/windows


TYPE-2 HYPERVISOR (Hosted hypervisor) ====>>
1.type-2 hypervisor is also called as hosted hypervisor.
2.its a hypervisor that runs on the host O.S. also known as os-level virtulization.
3.it does not have the direct access to the host hardware and resources.
4.less secure ( if there is any error in the hardware or the O.S then the whole VMs can crash )
3.example:- (oracle VirtualBox, workstation)

Architecture:-

Hardware ========> Host O.S ========> Hypervisor =======> VM =========> O.S ==========> ubuntu/centos/windows

MIGRATION =====>>>
    migration is the technique of moving a virtual machine from one host to another host or from one datastore to another datastore.

Types of migration:-
1.cold migration
2.vmotion (Live Migration)
3.suspended migration 
4.physical to virtual (P2V)
5.virtual to virtual (V2V)

Cold migration:-
1.The virtual machine move from one host to another host in powered-off state.
2.so the chances of failure in the cold migration is less than the other migration technique.

Suspended migration:-
1.migrating the virtual machine that is in suspended state/paused state.
2.VM can resume from the same point later.

Vmotion:-
1.Migrates a VM that is powered on.
2.it is also known as live migration.
3.No downtime for the VM.

Physical to Virtual (P2V):-
1.Converts a physical machine into a virtual machine.

Virtual to Virtual (V2V):-
1.Similar to P2V, but the source machine is already virtual.

 1.rsync is used to transfer and synchronize files/directories between local to server, server to server & server to local

commands to learn ====>>>

1. awk â†’ Process and extract columns from text

Print entire file:
awk '{print}' file.txt

Print first column:
awk '{print $1}' file.txt

Print first & third column:
awk '{print $1, $3}' file.txt

Print lines where column 3 > 100:
awk '$3 > 100 {print $0}' file.txt

Use with delimiter (e.g., CSV with commas):
awk -F, '{print $1,$2}' data.csv

--------------------------------------------------------------------

2. sed â†’ Stream editor (find & replace, delete, modify text)

Replace first occurrence of "cat" with "dog":
sed 's/cat/dog/' file.txt

Replace all occurrences:
sed 's/cat/dog/g' file.txt

Replace in file (in-place):
sed -i 's/cat/dog/g' file.txt

Delete line 5:
sed '5d' file.txt

Delete lines 2 to 4:
sed '2,4d' file.txt

--------------------------------------------------------------------

3. grep â†’ Search text patterns in files

Search for a word in a file:
grep "error" logfile.txt

Case-insensitive search:
grep -i "error" logfile.txt

Show line numbers:
grep -n "error" logfile.txt

Recursive search in directory:
grep -r "error" /var/log/

Show number of lines:
grep -c "error" logfile.txt

REDIRECTORS:-

Differentiate between the MBR and GPT

ðŸŸ¢ MBR (Master Boot Record)
1.Supports up to 4 primary partitions only.
2.Works with disks of size up to 2 TB.
3.Stores partition and boot data in the first sector of the disk.
4.Older technology, used in BIOS-based systems.

ðŸ”µ GPT (GUID Partition Table)
1.Supports 128 partitions (no limit like MBR).
2.Works with disks of size larger than 2 TB.
3.Stores multiple copies of partition data for backup and recovery.
4.Newer technology, used in UEFI-based systems.

   3.Not routable on the internet directly (requires NAT to communicate outside).
   4.Free of cost (anyone can use private IP ranges).

2.its based on client-server model
3.DHCP is an Application Layer protocol.

1.IP address
2.subnet mask
3.domain name
4.default gateway
5.DNS server address
6.Wins server address
          

IT WORKS ON THE DORA'S PROCESS =====>>>

Discover â€“----> The client broadcasts a request asking â€œIs there any DHCP server available to give me an IP?â€
Offer â€“-------> The DHCP server replies with an offer (IP address + subnet mask + gateway + DNS info).
Request ------> The client sends back a request saying â€œI want to use this IP, please confirm.â€
Acknowledge --> The DHCP server confirms and leases the IP to the client.

Configuration file====> { /etc/dhcp/dhcpd.conf }

SSL / TLS / HTTPS


ssl -----> secure socket layer
tls -----> transport layer security
https ---> hyper text transport protocol secure

HTTPS = HTTP + TLS ====>>> Secure website

---------------------------------------------------------------------------------

WHY WE NEED SSL/TLS:-
When you open a website and send data (password, OTP, card details)...
  1.Without SSL/TLS ===>>> data travels plain text, hackers can read it.
  2.With SSL/TLS ======>>> data becomes encrypted, no one can read it.

---------------------------------------------------------------------------------

HOW SSL/TLS WORKS:-

Symmetric:-
  1.faster and more efficient, suitable for large data.
  2.challenging key distribution, single key must be kept secret.

Asymmetric:-
  1.slower, suitable for small data.
  2.simplified key distribution, secure even if the public key is sharing.
  3.two keys: 1.public-key
            2.private-key

---------------------------------------------------------------------------------

WHAT IS SSL-CERTIFICATE:-
 An SSL certificate is a digital ID card of a website.

It contains:
  1.Website name (domain)
  2.Owner details
  3.Public Key
  4.Validity (expiry date)
  5.Digital Signature from CA (Certificate Authority)

---------------------------------------------------------------------------------

WORKING OF SSL-TLS:-
1.TCP handshake
2.certficate checking
3.key exchange
4.data exchange

1.Client Hello (Browser â†’ Server)
  Your browser says:
  "Hello server, I support these TLS versions and ciphers. Let's start secure connection!"

2.Server Hello (Server â†’ Browser)
  Server replies:
  "Hello! I choose this TLS version and cipher."

3.Server Sends SSL Certificate
  The server sends its certificate + public key.

4.Browser Verifies Certificate
  Browser checks:
  Was certificate signed by a trusted CA?
  Is the certificate expired?
  Is this certificate really for this website?
  If everything is OK â†’ Lock icon appears ðŸ”’

5.Key Exchange (Very Important)
  Browser creates a pre-master secret (random string) and:
  âœ”ï¸ Encrypts it with Serverâ€™s Public Key
  âœ”ï¸ Sends it to the server
  Only the server can decrypt it using its Private Key.
  Now both browser and server have the same secret.

6.Session Key Generation (Symmetric Key)
  Using the pre-master secret: 
  âœ”ï¸ Both sides generate same symmetric key 
  âœ”ï¸ This key is used for all further data communication
  Symmetric is fast â†’ perfect for actual data.

7.Encrypted Data Exchange (Real Communication)
  Now everything you send:
  Passwords
  OTP
  Messages
  Payments
  â†’ are encrypted with symmetric key â†’ super fast and secure.

---------------------------------------------------------------------------------

SIMPLY SAY...

When you open an HTTPS website:
Browser and server say "hello".
Server shows certificate.
Browser verifies it.
Browser creates a secret key.
Secret key is sent encrypted using asymmetric encryption.
Both sides generate a symmetric key.
All data now uses fast symmetric encryption.
âœ”ï¸ Data is secure
âœ”ï¸ Hackers cannot read it
âœ”ï¸ Website is trusted
âœ”ï¸ You see the lock ðŸ”’

â€œTLS uses asymmetric encryption for key exchange and symmetric encryption for data exchange.â€
â€œHTTPS is just HTTP running inside a TLS-secured tunnel.â€

WHY HASHING IS USED IN PASSWORDS:-

We donâ€™t store passwords.

Example:
